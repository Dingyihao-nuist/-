# /comments-check — 代码注释质量检查

对代码进行三维度的注释质量审查，输出违规清单和修复建议。

## 检查维度

### 维度 1：注释覆盖率 — 30% 最低标准

**规则：** 每 10 行有效代码中，至少应有 3 行注释。注释率 = 注释行数 / (代码行数 + 注释行数) ≥ 30%。

**统计口径：**
- **注释行**：`#` `//` `/* */` `/** */` `'''` `"""` 开头的行，以及行内注释 `// ...` `# ...`
- **有效代码行**：排除空行、纯花括号/括号行、只有 `else:` / `{` / `}` / `);` 的行
- **不计入**：文件头部的 license 注释、自动生成代码、import/export 语句后的 `// @ts-ignore` 等抑制注释

**检查重点：**
- ❌ 函数/方法完全无注释 → 严重违规
- ❌ 核心算法逻辑无行内注释 → 严重违规
- ❌ 复杂正则/条件判断无解释 → 中等问题
- ⚠️ 类/模块缺少 docstring → 轻微问题
- ⚠️ 公共 API 接口缺少参数说明 → 轻微问题

### 维度 2：注释与代码匹配度

**规则：** 注释描述必须与代码实际行为一致。过时、错误、误导性注释视为严重缺陷。

**检查方法：**
1. **语义对比** — 阅读注释描述的意图，对照代码实际逻辑
2. **命名一致性检查** — 注释中提到的变量名/函数名是否在代码中存在
3. **返回值匹配** — 注释声明的返回值类型/含义是否与代码实际返回一致
4. **条件逻辑验证** — 注释说"A 情况下执行 B"，代码是否确实如此
5. **副作用匹配** — 注释声明的副作用（修改状态、写文件）是否属实

**常见问题：**
- ❌ 代码重构后注释未更新（函数签名已变，注释仍是旧版）
- ❌ 注释说"返回 True"但实际返回了字符串
- ❌ 注释写"异步执行"但代码是同步的
- ❌ 复制粘贴的注释，与当前代码逻辑无关
- ❌ TODO/FIXME 注释已过期（功能已实现但标记未删除）
- ⚠️ 注释的参数名与实际参数名不一致
- ⚠️ 注释中的数据类型与实际类型不匹配

### 维度 3：初学者可读性

**规则：** 注释应面向"刚入门的新人"，避免假定读者具备领域知识。用平实的中文解释"为什么这样做"，而不仅仅是"做了什么"。

**评分标准：**

| 等级 | 标准 |
|------|------|
| **好** | 解释了业务背景 + 为什么这样写 + 关键参数含义，新手无需查文档就能理解 |
| **可接受** | 至少说明了函数用途和参数的基本含义 |
| **差** | 只写了"初始化"、"处理数据"、"获取列表"等废话注释 |
| **致命** | 无注释，或用英文/专业术语堆砌，新手完全看不懂 |

**具体要求：**
- ✅ 解释业务含义："为什么需要这个判断？这个阈值为什么是 0.5？"
- ✅ 说明特殊场景：边界条件、异常处理的原因
- ✅ 展开缩写和专业术语：`RAG` → `检索增强生成(Retrieval-Augmented Generation)`
- ✅ 复杂操作的步骤拆解：用 `Step 1/2/3` 标注流程
- ❌ 废话注释：`count += 1  // 自增1`（代码本身已经清楚）
- ❌ 纯英文注释（项目面向中文开发者时）
- ❌ 过度使用专业术语不加解释

## 工作流程

### 输入
- 用户指定要检查的文件、目录或 git diff
- 默认检查当前分支相对于 main 的所有变更文件

### 执行步骤

1. **确定检查范围** — 列出所有待检查的源文件（`.py` `.js` `.jsx` `.ts` `.tsx` `.vue` `.java` `.go`）
2. **逐文件分析** — 对每个文件执行三维度检查
3. **生成报告** — 按严重程度列出所有问题，附带具体行号和修复建议

### 输出格式

```
## 注释检查报告

**检查范围：** 3 个文件，共 245 行代码
**总体评分：** 注释率 18% ❌ | 匹配度 85% ⚠️ | 可读性 中

### 🔴 严重问题（必须修复）

| # | 文件:行号 | 维度 | 问题描述 | 修复建议 |
|---|-----------|------|----------|----------|
| 1 | config.py:15 | 覆盖率 | `load_config()` 函数 22 行无任何注释 | 添加 docstring + 关键步骤行内注释 |
| 2 | chain.py:42 | 匹配度 | 注释说"返回 dict"但实际返回 `str` | 更新注释为"返回 JSON 字符串" |
| 3 | embeddings.py:8 | 可读性 | 变量名 `bge` 未解释 | 注释说明 `bge = BAAI General Embedding` |

| # | 文件:行号 | 维度 | 问题描述 |

| # | 文件:行号 | 维度 | 问题描述 |

```

## 检查清单（执行时逐条核对）

### 覆盖率
- [ ] 每个函数是否有 docstring/JSDoc
- [ ] 每个类是否有类级别注释
- [ ] 核心算法/业务逻辑是否有行内注释
- [ ] 非显而易见的常量/魔法数字是否有解释
- [ ] 正则表达式是否有分解说明
- [ ] 状态机/复杂条件判断是否有流程注释

### 匹配度
- [ ] docstring 中的参数名是否与签名一致
- [ ] 注释的返回值类型是否与代码实际返回匹配
- [ ] 注释描述的功能是否与代码逻辑一致
- [ ] 是否存在代码已修改但注释未更新的情况
- [ ] TODO/FIXME 是否已过期

### 可读性
- [ ] 是否用中文解释了业务背景
- [ ] 专业术语是否有展开说明
- [ ] 是否解释了"为什么"而不只是"是什么"
- [ ] 边界条件和特殊处理是否有说明
- [ ] 新手能否仅凭注释理解代码意图
